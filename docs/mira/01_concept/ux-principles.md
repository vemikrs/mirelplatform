# Mira UX Principles

Mira の UX は「軽く、自然で、正確」であることを基盤に設計する。

---

## Principle 1: Always Understand Where You Are
ユーザがどの画面にいるか、何をしようとしているかを理解したうえで回答する。

- コンテキストヘルプ  
- 操作対象に応じた回答  
- ロールベースの説明差分  

「読み解く手間」を減らす。

---

## Principle 2: Show the Next Action
ユーザが次に何をすべきかを常に明確に示す。

- 「この画面でできること」  
- 「設定の次のステップ」  
- 「解決のための UI パス」  

案内は具体的で、迷わせない。

---

## Principle 3: Minimize Cognitive Load
情報量を絞り、段階的に開示する。

- 結論 → 理由 → 詳細  
- 要約の提示  
- 過剰な専門用語の抑制  

理解負荷の軽減が UX の中心となる。

---

## Principle 4: Consistent Interaction Model
どのアプリでも同じ操作感で Mira を呼び出せるようにする。

- 右サイドパネル  
- ショートカットキー  
- 「?」アイコンでの起動

一貫した操作体系は信頼につながる。

---

## Principle 5: Assist, Never Interrupt
ユーザの作業の流れを邪魔しない。

- 非モーダル優先  
- 必要な説明だけ  
- AI 側からの過度な提案は抑制  

支援は自然に、控えめに。

---

## Principle 6: Transparent Reasoning
回答の根拠を明らかにし、曖昧さを排除する。

- 「なぜこの操作か」  
- 「なぜこのエラーが起きたか」  
- 「判断の理由」  

透明性は信頼性と学習効率を高める。
