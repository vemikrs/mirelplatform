# mirel Studio LP コピー

> モデルが、業務アプリケーションのすべてを形づくる。

---

## Hero Message

**業務の構造を記述するだけで、アプリケーションが成立し、運用と改善が続いていく。**

mirel Studio は、現場の業務を **"モデルとして扱える形"** へ変換する新しいアプリケーション設計基盤です。

---

## Why mirel Studio?

### 業務は「画面」ではなく「構造」から始まる。

従来のローコード/ノーコードは UI → データ の順でアプリを組み立てます。しかし複雑な業務はそれでは破綻します。

mirel Studio は、**業務構造（モデル） → UI/ロジック/データ → 運用** という一貫した原則に基づき設計されています。

---

## Key Features

### 1. Modeler（mirel Studio Modeler）

#### 業務の構造をそのまま表現できるモデルエディタ

- 階層的なドメイン構造
- 複合エンティティ
- 関係・依存の可視化
- 型に基づいた整合性管理

**業務ロジックを "構造" の単位で扱える** LCAP は他にありません。

---

### 2. Form Designer（mirel Studio Form Designer）

#### モデルから自然に導かれる UI

- ドラッグ＆ドロップでレイアウト
- 型に応じたコンポーネント候補
- 表示条件・バリデーション
- モデル変更との自動同期

**UI はモデルの表現であり、個別実装の集合ではありません。**

---

### 3. Flow Designer（mirel Studio Flow Designer）

#### 業務を動かす「振る舞い」を直感的に設計

- データイベント（onCreate/onUpdate/onDelete）
- フォームイベント
- 定期実行（cron）
- 条件分岐・外部 API 連携

業務に必要なルールや判断を **視覚的に、整合性を保ったまま** 定義できます。

---

### 4. Data Browser（mirel Studio Data Browser）

#### データの観測と軽微な操作を安全に

- スプレッドシートのような一覧
- インライン編集（権限付き）
- Saved View（条件セットの保存）
- CSV/Excel 出力

業務の状態を把握し、即時改善につなげるための環境です。

---

### 5. Release Center（mirel Studio Release Center）

#### 変更が秩序として蓄積される仕組み

- Draft → Release → Deploy
- 差分検出
- ロールバック
- 環境ごとの反映管理

業務アプリケーション特有の **「日々の改善」と「安定稼働」** を両立させます。

| 一般的 LCAP | mirel Studio |
|------------|--------------|
