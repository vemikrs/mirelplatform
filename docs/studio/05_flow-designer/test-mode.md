# mirel Studio Flow Designer テスト実行モード

> 公開前のフロー検証を可能にするテストモード

---

## 1. テスト実行の目的

- 条件式の誤り検出
- API / Script 実行の確認
- 分岐経路の可視化
- 実行結果のログ確認

---

## 2. テストモードの機能

### 2.1 シミュレーション実行

- Form の入力値を模擬データとして投入
- Flow のノードを順次実行
- 分岐結果を表示

### 2.2 実行ログ

```
[Start]
  → validateFields
  → decision(customer.age >= 20) = true
  → sendMail
  → End
```

### 2.3 ブレークポイント

任意のノードで停止可能。デバッグ目的で使用。

---

## 3. モックデータ

| 対象 | モック内容 |
|------|-----------|
| API 呼び出し | レスポンスのモック |
| Script | モック引数 |
| 外部サービス | 即時応答により高速テスト |

---

## 4. UI 構成

```
┌────────────────────────────────────────┐
│ 画面左：Flow 図                        │
│ 画面右：ログ・テスト設定              │
│ 下部：モックデータエディタ            │
└────────────────────────────────────────┘
```

---

## 5. 制限事項

| 制限 | 説明 |
|------|------|
| 外部サービス連携 | "実行なし"のサンドボックス |
| 大量データ処理 | 擬似実行 |
| 本番データ | アクセス不可 |

---

## 6. 設計意図

- Builder が自ら検証可能
- 本番 Deploy までの失敗を減らす
- Flow 開発体験の向上
