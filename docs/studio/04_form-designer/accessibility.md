# mirel Studio Form Designer アクセシビリティ方針

> 生成フォームにおけるアクセシビリティの標準化

---

## 1. アクセシビリティの前提方針

mirel Studio Form Designer は生成フォームのアクセシビリティを重視し、以下の方針を基盤とする。

- **「操作説明が不要であること」を最優先**
- 読み上げ・キーボード操作などを考慮
- 標準コンポーネントの利用とガイドライン統一
- W3C WCAG の基本要件を満たす

---

## 2. 入力コンポーネントのルール

### 2.1 ラベルと input の紐づけ

```html
<label for="customer-name">顧客名</label>
<input id="customer-name" type="text" />
```

- `label for=xxx` を必ず生成
- フィールド名が未設定の場合、FieldId を用いる

### 2.2 エラーメッセージ

| ルール | 説明 |
|--------|------|
| 色のみに依存しない | 形状・アイコンを併用 |
| 各項目直下に表示 | 画面下部にまとめない |
| aria 属性を付与 | 読み上げ対応 |

---

## 3. キーボード操作の保証

| 操作 | サポート内容 |
|------|-------------|
| Tab 移動 | Form Designer の要素順に従う |
| ドロップダウン | `↑↓ Enter Esc` を必ず実装 |
| モーダル操作 | キーボード対応必須 |
| フォーカス | 視覚的に明確なフォーカスリング |

---

## 4. カラーデザイン

| 項目 | ガイドライン |
|------|-------------|
| 重要箇所 | 赤を単独で使用しない（色弱配慮） |
| コントラスト比 | 4.5:1 以上 |
| 意味の伝達 | 色ではなく「形・枠線」で表現 |

---

## 5. レスポンシブ対応

- 画面幅 480px 以下で 1 カラムへ自動切替
- 手動設定がある場合でも「強制崩れ防止ロジック」を持つ
- モバイルでのタッチ操作を考慮

---

## 6. 障害ポイントの自動検出（将来拡張）

| 検出対象 | 説明 |
|---------|------|
| ラベル欠損 | `label` 未設定を検出 |
| コントラスト比不足 | 視認性の問題を可視化 |
| 色依存 UI | 色のみに依存した表現を警告 |

---

## 7. 設計意図

- 生成アプリの利用障壁を下げる
- 誰でも同じ操作体験ができるようにする
- 障害者差別解消法および W3C WCAG の基本要件を満たす
