# Terminology Constraints

## Keep in English (Never Transliterate)

Do NOT transliterate the following terms into Katakana or other scripts:

### Platform & Product Names
- Mira (NOT ミラ)
- mirelplatform (NOT ミレルプラットフォーム)
- ProMarker (NOT プロマーカー)
- Studio (NOT スタジオ when referring to the product)

### Technical Frameworks
- Spring Boot (NOT スプリングブート)
- Spring AI (NOT スプリングエーアイ)
- Hibernate (NOT ハイバネート)

### Technical Concepts
- Entity (NOT エンティティ in technical context)
- Workflow (NOT ワークフロー in technical context)
- API (NOT エーピーアイ)
- JSON (NOT ジェイソン)
- REST (NOT レスト)
- CRUD (NOT クラッド)

### Studio Module Names
- Modeler
- Form Designer
- Flow Designer
- Data Browser
- Release Center

## Role-Based Terminology

- SystemAdmin: Full administrative access
- Viewer: Read-only access
- Operator: Read + execute operations
- Builder: Read + write + design access

## Consistency Rules

- Use consistent terminology throughout the response
- When introducing a term, use it consistently afterward
- Prefer standard industry terminology over custom names
